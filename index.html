Bot Overview

The Options Trading Bot is an automated trading system designed for the Upstox brokerage platform. It implements a VWAP-RSI crossover strategy for options trading with integrated risk management, real-time data streaming, and automated order execution.

Important Disclaimer

This bot is for educational and demonstration purposes only. Trading involves substantial risk of loss and is not suitable for every investor. Always test strategies in a simulated environment before using real funds.

Key Features

  • Real-time market data streaming via WebSocket
  • Automated VWAP-RSI crossover strategy for entry signals
  • Integrated GTT (Good Till Triggered) orders with stop loss and target
  • Trailing stop loss functionality
  • Position management with automatic exit on conditions
  • Windows wake lock to prevent system sleep during trading
  • Audio alerts for key events
  • Emergency kill switch with keyboard controls
  • Trade cooldown and daily trade limit management

Setup & Installation

Prerequisites

  • Python 3.8 or higher
  • Upstox trading account with API access enabled
  • Windows OS (for wake lock functionality)
  • Required Python packages (install via pip)
pip install upstox-client keyboard pandas numpy pytz python-dotenv

Initial Configuration

  1. Clone or download the bot source code
  2. Create a .env file in the project root with your Upstox API credentials
  3. Run the bot for the first time to authenticate with Upstox
  4. Configure trading parameters in the Config class based on your risk tolerance
# .env file template
api_key=your_api_key_here
api_secret=your_api_secret_here
redirect_url=https://your-redirect-url.com
state=random_state_string
access_token=

On first run, the bot will open a browser window for Upstox OAuth authentication. After logging in, copy the redirect URL from your browser and paste it into the terminal to complete the authentication process.

Adjust SL_POINTS, TARGET_POINTS, and LOTS according to your risk tolerance and account size. The default values are for demonstration purposes only.

System Components

The bot is built using a modular architecture with the following key components:

Authenticator

Handles OAuth authentication with Upstox API, token management, and validation.
  • get_access_token() - Retrieves valid access token
  • check_token_validity() - Validates token expiration
  • generate_access_token() - Generates new token via OAuth flow

Data_Processor

Processes market data, converts tick data to candles, and manages instrument keys.
  • get_instrument_key() - Finds instrument key by symbol
  • get_option_key() - Calculates option key based on strike
  • convert_to_candles() - Aggregates ticks into candles

Data_Collector

Fetches market data from Upstox API including historical, intraday, and real-time data.
  • get_historic_data() - Fetches historical candle data
  • get_intraday_data() - Fetches intraday candle data
  • get_option_price() - Gets latest option price

Calculations

Static methods for technical indicators, risk calculations, and trade parameters.
  • calculate_vwap() - Calculates Volume Weighted Average Price
  • calculate_rsi() - Calculates Relative Strength Index
  • calculate_quantity() - Calculates order quantity

Strategies

Contains the core VWAP-RSI trading strategy and trade validation logic.
  • vwap_rsi_strategy() - Implements the trading strategy
  • pre_check_validation() - Validates trade conditions

Terminator

Manages graceful shutdown and emergency kill functionality via keyboard shortcuts.
  • kill_bot() - Graceful shutdown
  • emergency_kill() - Immediate termination

Trading Strategy

VWAP-RSI Crossover Strategy

The bot implements a dual-timeframe VWAP (Volume Weighted Average Price) and RSI (Relative Strength Index) crossover strategy for options trading.

Call Entry Conditions (CE)

Stage 1 (Immediate Entry): Current candle closes above VWAP AND RSI crosses above 50

Stage 2 (Delayed Entry): Previous candle closed above VWAP, current RSI > 50, and previous RSI was between 40-50

Put Entry Conditions (PE)

Stage 1 (Immediate Entry): Current candle closes below VWAP AND RSI crosses below 50

Stage 2 (Delayed Entry): Previous candle closed below VWAP, current RSI < 50, and previous RSI was between 50-60

Risk Management

  • Stop Loss: Fixed at entry price minus SL_POINTS (default: 13 points)
  • Target: Fixed at entry price plus TARGET_POINTS (default: 100 points)
  • Trailing Stop: Activated after certain profit, trails by TRAILING_GAP (default: 10 points)
  • Trade Limits: Maximum TRADE_LIMIT (default: 5) trades per day
  • Cooldown: ENTRY_COOLDOWN (default: 10 seconds) between trades

Position Sizing

The bot calculates position size based on:

  • Lot size of the underlying instrument
  • Config.LOTS parameter (default: 1)
  • Quantity = Lot Size × LOTS

Bot Workflow

Startup Sequence

  1. Load configuration from .env file and Config class
  2. Authenticate with Upstox API (OAuth flow if needed)
  3. Initialize data processors and load instrument data
  4. Fetch initial historical and intraday data
  5. Wait for 3-minute candle mark and market open time
  6. Establish WebSocket connection for real-time data
  7. Start listening for keyboard termination commands

Trading Cycle

  1. Receive real-time tick data via WebSocket
  2. Aggregate ticks into 1-minute and 3-minute candles
  3. Calculate VWAP and RSI indicators on 3-minute timeframe
  4. Check for VWAP-RSI crossover signals
  5. Validate trade conditions (market hours, cooldown, trade limit)
  6. Calculate appropriate strike price and option instrument key
  7. Place GTT order with entry, stop loss, and target levels
  8. Monitor position and manage exit (stop loss, target, or trailing stop)
  9. Clean up after trade exit and resume monitoring for new signals

Controls & Safety Features

Keyboard Controls

'Q' Key - Graceful Shutdown

Pressing 'Q' will:

  • Check for open positions and exit them if present
  • Disconnect from WebSocket gracefully
  • Log shutdown message and exit the program

'ESC' Key - Emergency Kill

Pressing 'ESC' will:

  • Immediately disconnect from WebSocket
  • Force exit the program without position checks
  • Log critical emergency stop message

Audio Alerts

The bot provides audio feedback for key events using system sounds:

  • WebSocket Connected: High-pitched beep (2000Hz, 200ms)
  • WebSocket Error: Series of low beeps (400Hz)
  • WebSocket Disconnected: Medium-pitched long beep (1200Hz, 1000ms)
  • Trade Entered: Medium-high beep (900Hz, 200ms)
  • Trade Exited: Medium-low beep (700Hz, 200ms)
  • Error: Long error beep (1000Hz, 1000ms)

Wake Lock (Windows Only)

On Windows systems, the bot activates a wake lock to prevent the system from going to sleep during trading hours. This ensures uninterrupted operation during market hours.

Platform Compatibility Note

The wake lock feature is only available on Windows. On other operating systems, ensure your system's sleep settings are configured to prevent automatic sleep during trading hours.
